## English

### 🎯 Mission
Max token compression, minimal performance loss.

### 🔬 Research Backing
| Method | Origin | Compression | Performance Loss |
|--------|--------|-------------|------------------|
| LLMLingua | Microsoft | 20x | < 2% |
| ProCut | EMNLP 2025 | 8x | < 3% |
| 500xCompressor | Stanford | 6-480x | < 5% |
| Granularity Control | Amazon Science | 4x | **+23% GAIN** |

### 🚀 Activation
- User says: "抠门", "save tokens", "reduce costs", "optimize prompts"
- All cost-sensitive LLM operations
- Budget-constrained scenarios

### 📊 Differentiated Compression
| Component | Rate | Priority |
|-----------|------|----------|
| Core Instructions | 0-10% | CRITICAL |
| User Query | 0-10% | CRITICAL |
| System Prompt | 20-40% | HIGH |
| Few-shot Examples | 50-80% | LOW |
| Chitchat / Redundancy | 70-90% | LOWEST |

### ⚡ Methods
1. **Attribution Pruning**: Remove tokens <5% output contribution
2. **Dense Rewrite**: No pleasantries, compact directives, model-native phrasing
3. **Output Control**: Minified code, direct answers, compact Chain-of-Thought
4. **Tool Batching**: Minimize round-trips, strip output, SearchCodebase first
5. **KV Cache**: Static prefix reuse = 50-90% cost reduction

### 🛡️ Performance Guards
✅ Never mutilate core instructions  
✅ Anchor all entities/numbers/identifiers  
✅ Preserve semantic integrity  
✅ Capability retention > perfect compression

## 中文

### 🎯 使命
极致压缩，性能无损。抠到每一个token。

### 🔬 学术背书
| 方法 | 来源 | 压缩比 | 性能损失 |
|------|------|--------|----------|
| LLMLingua | 微软 | 20倍 | < 2% |
| ProCut | EMNLP 2025 | 8倍 | < 3% |
| 500xCompressor | 斯坦福 | 6-480倍 | < 5% |
| 粒度控制 | 亚马逊 | 4倍 | **+23% 效果提升** |

### 🚀 激活条件
- 用户说: "抠门", "省token", "降本", "优化prompt"
- 所有成本敏感场景
- 预算受限环境

### 📊 差异化压缩
| 组件 | 压缩率 | 优先级 |
|------|--------|--------|
| 核心指令 | 0-10% | 致命 |
| 用户问题 | 0-10% | 致命 |
| System Prompt | 20-40% | 高 |
| 示例样本 | 50-80% | 低 |
| 冗余闲聊 | 70-90% | 最低 |

### ⚡ 方法论
1. **归因剪枝**: 删除对输出贡献<5%的token
2. **致密改写**: 无客套话, 紧凑指令, 模型原生表达
3. **输出控制**: 极简代码, 直接答案, 压缩思考链
4. **工具批处理**: 最少交互回合, 裁剪输出, 优先SearchCodebase
5. **KV缓存**: 静态前缀复用 = 省50-90%费用

### 🛡️ 性能保障
✅ 绝不阉割核心指令  
✅ 锚定所有实体/数字/标识  
✅ 保证语义完整性  
✅ 能力保留 > 完美压缩
